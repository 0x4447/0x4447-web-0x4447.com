Our Open Source Tools

We love efficiency and hate doing repetitive work, that is why we build tools to help us do the repetitive work while also sharing this tools to the world so others can also benefit. We strongly believe that knowledge should be shared, and presented in the simplest way possible to let other learn. By sharing we can help others have new ideas the will benefit our community.

Potato

The purpose of the Potato CLI tool is to streamline the process when you want to host a static web page on S3 and deliver it using CloudFront.

Tomato

This is our own take on the express-generator CLI that comes with ExpressJS. We made this project because we always spend quite some time brining the default ExpressJS template up to speed. Especially when we have to constantly spin up new micro-services.

A nice feature of this project is the simplicity on how you can customize it yourself. If you have your own style, you can just clone this repo and edit the source folder to your liking.

Cucumber

This is a very simple and small project that we created to save some time while working with Heroku. This way we can easily auto generate the .env file when we develop the project locally.

Hot Pepper

If you work with NodeJS projects and prefer to run them on a Linux box using SystemD as the process manager. Then this small project will help you save a lot of time by creating the systemD .service file for you based on the content of the package.json file and the location in which the CLI was executed.

Broccoli

If you are a developer with a local Nginx setup, and like to give the sites that you work on, a local domain name, then for sure you experienced the tedious work of creating a Nginx config file and restart Nginx to load the new configuration. This CLI will do it all for you automatically.
